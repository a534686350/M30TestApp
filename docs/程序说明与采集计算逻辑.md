# M30TestApp.V2 程序说明与采集计算逻辑

本文档说明 M30TestApp.V2 的程序定位、功能模块、实现结构、设备通讯方式、自动测试流程、板卡采集逻辑、指标计算方式、异常处理和主要界面功能。

## 1. 程序定位

M30TestApp.V2 是用于 M30 压力传感器测试的 WPF 上位机程序，基于 .NET 8、WPF、MVVM 架构实现。程序目标是替代旧版 WinForms/ASLab 测试软件，保留压力控制、温度控制、板卡采集、工位映射、测试流程、数据矩阵、指标计算、CSV 导出和断点续测能力。

程序支持两类后端：

- SIM：模拟设备后端，用于无硬件调试、界面验证、流程验证。
- HW：真实硬件后端，用于连接压力控制器、烘箱、采集板、DMM/切换单元、电源和通道板。

## 2. 解决方案结构

```text
M30TestApp.V2
├─ src/M30TestApp.Core
│  ├─ Common        路径、日志、全局偏好、设备总线
│  ├─ Config        INI、Command.ini、工位表、测试方案、压力类型
│  ├─ Devices       设备接口、SIM 实现、HW 实现、设备工厂
│  ├─ TaskScript    测试脚本解析、任务调度、内置 Action
│  ├─ Data          数据矩阵、指标计算、CSV 导出、断点保存
│  └─ TestSession   会话对象，组装设备、方案、矩阵和任务运行器
│
└─ src/M30TestApp.Wpf
   ├─ Views         WPF 页面和自定义矩阵控件
   ├─ ViewModels    主窗口、实时测试、手动调试、配置、日志、设置
   ├─ Mvvm          ViewModelBase、RelayCommand、AsyncRelayCommand
   ├─ Themes        浅色/深色主题
   └─ Converters    XAML 转换器
```

核心原则是：业务逻辑和设备通讯放在 `M30TestApp.Core`，界面绑定和操作入口放在 `M30TestApp.Wpf`。

## 3. 主界面功能

### 3.1 顶部状态栏

顶部显示当前测试站、测试方案、运行状态和设备在线状态。

当前显示的设备状态包括：

- 压控：压力控制器。
- 烘箱：温度控制设备。
- 切换单元：DMM/继电器切换单元。
- 板卡：M30-DAC 数据采集板。
- 电源：激励电源。
- 通道板：串口通道控制板。

状态由设备的 `ConnectionState` 驱动，显示为在线、连接中、离线或故障。

### 3.2 实时测试页

实时测试页是自动测试主页面，包含：

- KPI：工位总数、合格数、警告数、异常数、当前温度点、当前压力点。
- 运行控制：开始、停止、进度条、当前步骤、最后错误。
- 板卡采集数据矩阵：按工位和测试点显示实时采集值。
- 测试统计：合格、警告、异常百分比。
- 实时日志：显示测试执行过程和设备动作。

板卡采集数据矩阵支持像表格一样选中、全选和复制选中数据，复制时包含表头，便于粘贴到 Excel 或其他表格工具中分析。

### 3.3 手动调试页

手动调试页用于硬件联调和单项操作，包含：

- 采集卡串口打开/关闭。
- 烘箱串口打开/关闭、设温、读温、停止。
- 压力控制器连接、加压、泄压、读压、读量程、读状态、切换表压/绝压。
- 电源电压切换。
- UT 电源继电器切换。
- 单工位采集和批量采集。
- TX/RX 原始报文监视。
- 操作历史记录。
- 批量采集结果导出 CSV。

手动页的表压/绝压切换会真实调用压力控制器 `SetPressureTypeAsync`，不再只是界面模拟日志。执行加压前也会根据当前测量模式先下发压力类型。

### 3.4 配置页

配置页用于维护运行参数，包含：

- 方案：测试方案文件夹、传感器型号、压力点、温度点、默认压力类型。
- 指标限值：Offset、Span、Linearity、TCO、TCS、TCR、THO、THS、PH、TCT 等上下限。
- 参数控制：设备启用、延迟参数、压力指令模板。
- 设备：压力控制器、烘箱、切换单元、采集卡、电源、通道板参数。
- 指令：Command.ini 中的设备指令模板。
- 工位：工位对应表、扫码录入、批量生成、导入导出。
- 测试流程：TaskScript 步骤编辑。
- 计算：参与合格判定的指标开关。
- 版本信息。
- 系统设置：主题、日志保留、自动加载上次方案、异常中断保存断点等。

### 3.5 日志页

日志页显示应用级结构化日志。日志写入采用后台队列，避免测试过程中频繁写磁盘导致采集线程或 UI 卡顿。

## 4. 配置文件

### 4.1 Setting.ini

`Setting.ini` 保存运行配置，典型内容包括：

- `[DefaultLoadClass]`：各设备使用 SIM 还是 HW 后端。
- `[Device.*]`：设备型号、地址、串口参数、GPIB 地址。
- `[SwitchUnitCards]`：板卡对应切换单元通道。
- `[ValveSettings]`：阀门通道映射。
- `[DelaySettings]`：继电器切换、采集间隔、烘箱保温、泄压等待等延迟参数。
- `[Preferences]`：主题、日志保留、自动加载方案、断点保存等偏好。

### 4.2 Command.ini

`Command.ini` 保存不同设备型号的指令模板。压力控制器关键指令包括：

- `Open`
- `MachineType`
- `UpperLimit`
- `SetPressure`
- `Vent`
- `SetAbs`
- `SetGaug`
- `SetDiff`
- `ZeroCheck`
- `ReadPressure`
- `SetMeasure`
- `SelfTest`
- `ReadStatus`

程序通过 `CommandDictionary` 读取模板，并在需要时替换参数。

### 4.3 工位对应表

工位对应表定义每个 Slot 的序列号、阀位、板卡位、板卡工位号、夹具位、DMM 通道、阀门等映射。采集时程序根据工位找到板卡地址和通道地址。

### 4.4 测试方案 INI

测试方案保存：

- 传感器型号。
- 压力单位。
- 精度。
- 默认压力类型。
- 温度点。
- 压力点。
- 单点压力类型覆盖。
- 指标上下限。
- 指标启用开关。
- TaskScript 流程脚本。

## 5. 设备抽象

设备统一通过接口访问：

- `IPressureController`：压力控制器。
- `IOven`：烘箱。
- `IDmm`：DMM/切换单元。
- `IDac`：采集板。
- `IPowerSupply`：电源。
- `IBoard`：通道板。

每个设备都继承基本能力：

- `OpenAsync`
- `CloseAsync`
- `SelfTestAsync`
- `State`
- `StateChanged`

真实硬件实现位于 `Devices/Hw/HwDevices.cs`，模拟实现位于 `Devices/Sim/SimDevices.cs`。

## 6. 自动测试总流程

自动测试入口为实时测试页的开始按钮。

流程如下：

```text
TestRunView
  → TestRunViewModel.RunAsync
    → RunSetupWindow 选择方案和工位
    → TestSession.ApplyRunConfig
    → TestSession.RunAsync
      → TaskScript.Parse
      → TaskRunner.RunAsync
        → IAction.ExecuteAsync
          → 设备控制
          → 板卡采集
          → 数据矩阵写入
          → 指标计算
          → CSV 导出
```

完整性能测试通常使用：

```text
Run:PerformanceTest
```

该 Action 内部完成设备初始化、探漏、温度点循环、压力点循环、上行/下行压力采集、指标计算和数据保存。

## 7. TaskScript 说明

TaskScript 格式：

```text
模块:动作[,参数1[,参数2...]]|模块:动作...
```

常见命令：

| 命令 | 说明 |
| --- | --- |
| `Initial:Pressure` | 初始化压力控制器 |
| `Initial:Temp` | 初始化烘箱 |
| `Initial:Board` | 初始化通道板 |
| `Initial:DMM` | 初始化切换单元 |
| `Initial:CommuTest` | 通讯自检 |
| `DAQ:ClearData` | 清空采集矩阵 |
| `DAQ:TestType,<type>` | 设置测试类型标签 |
| `TP:SetPressurePoint,<idx>` | 切换到指定压力点 |
| `TP:SetTempPoint,<idx>` | 切换到指定温度点 |
| `Read:UT` | 采集 UT |
| `Read:Usource` | 采集 Usource |
| `Read:Isource` | 采集 Isource |
| `Read:Usign` | 采集 Usig |
| `Run:PerformanceTest` | 执行完整性能测试 |
| `Cal:Test` | 计算性能指标 |
| `Save:TestData` | 保存测试数据 |

当前运行时未知命令会直接报错，不再静默跳过，避免测试流程配置错误被忽略。

## 8. 板卡采集逻辑

板卡采集由 `DacBatchSampler.SampleAllAsync` 实现。

### 8.1 工位解析

每个工位通过 `SlotEntry` 表示。采集前程序根据工位号解析：

- 板卡地址。
- 板卡通道。
- 工位名称。
- 序列号。

如果工位表中找不到指定工位，程序会按默认布局公式生成临时工位映射，保证手动采集和批量采集仍可运行。

### 8.2 采集类型

支持的板卡采集类型：

- `Usig`：信号输出。
- `UT`：温度/UT 输出。
- `Usource`：激励电压。
- `Isource`：激励电流。

内部枚举为：

```text
DacMeasureKind.Usig
DacMeasureKind.UT
DacMeasureKind.Usource
DacMeasureKind.Isource
```

### 8.3 采集过程

批量采集过程：

```text
打开采集板
确保目标矩阵列存在
根据工位范围生成采集列表
逐工位：
  解析板卡地址和通道
  如果采集 UT：
    切换 UT 电源到当前板卡
    等待继电器稳定
  调用采集板读值
  写入 DataMatrix
  触发 UI 单元格刷新
  记录日志
  等待采集间隔
```

### 8.4 UT 电源切换

UT 采集前需要按板卡切换继电器。程序会：

1. 打开 DMM/切换单元。
2. 关闭当前板卡对应继电器通道。
3. 打开其他板卡对应通道。
4. 等待 `ValveSwitchMs`。

这样可以保证当前板卡 UT 供电状态正确。

### 8.5 采集延迟

延迟参数来自 `[DelaySettings]`：

- `UsigDelayMs`
- `UtDelayMs`
- `UsourceDelayMs`
- `IsourceDelayMs`
- `ValveSwitchMs`

手动批量采集也可以设置自定义采集间隔。

### 8.6 硬件报文

M30-DAC 硬件采集使用串口 Modbus 风格帧：

```text
[card, function, channel, crcLo, crcHi]
```

功能码：

- `2`：Usource
- `3`：Isource
- `4`：Usig
- `5`：UT
- `6`：切换 UT 电源

程序会校验 CRC，失败时重试，最终失败会写入错误日志并将对应单元格标记为异常。

## 9. 数据矩阵

数据矩阵由 `DataMatrix` 管理，是按工位和列名组织的稀疏矩阵。

行：

```text
Slot1
Slot2
...
Slot256
```

列：

```text
T1_UT
T1_USC
T1_ISC
T1P1_USG
T1P2_USG
T2P1_USG
Offset
Span
NL
...
```

每个单元格包含：

- Key：列名。
- Value：显示值。
- Status：Empty、Pending、Ok、Warn、Error。
- UpdatedAt：更新时间。

UI 通过 `CellUpdated` 事件实时刷新。实时测试页统计改为按行状态增量更新，减少大量采集时的 UI 计算开销。

## 10. 压力和温度控制逻辑

### 10.1 压力类型

支持：

- 表压：Gauge。
- 绝压：Absolute。
- 差压：Differential。

压力类型来自测试方案默认值，也可以被单个压力点覆盖。执行压力点前会调用：

```text
SetPressureTypeAsync
```

对应 Command.ini：

```text
SetGaug
SetAbs
SetDiff
```

### 10.2 设置压力

设置压力时程序会：

1. 切换压力类型。
2. 设置目标压力、单位和精度。
3. 进入压力控制模式。
4. 等待压力稳定或达到设定等待时间。

### 10.3 温度点

温度点执行时程序会：

1. 设置烘箱目标温度。
2. 等待温度达到目标。
3. 按配置保温。
4. 执行该温度点下的 UT、USC、ISC 和压力点 USG 采集。

## 11. 完整性能测试流程

`Run:PerformanceTest` 的主要流程：

```text
初始化设备
执行探漏
保存断点
循环温度点：
  设置温度
  保温
  采集 UT
  采集 Usource
  采集 Isource
  循环压力点：
    切换压力类型
    设置压力
    保压
    采集 Usig
    保存断点
  下行回差采集：
    从高压点回到低压点
    采集 Usig_R
保存矩阵
计算指标
导出 CSV
删除断点
关闭/泄压/下电
```

## 12. 指标计算方式

指标计算由 `MetricsCalculator.Calculate` 实现。计算要求至少 3 个温度点和 3 个压力点。

默认取：

- `t1`：第 1 个温度点。
- `t2`：第 2 个温度点。
- `t3`：第 3 个温度点。
- `p0`：第 1 个压力点。
- `p50`：中间压力点。
- `p100`：最后一个压力点。

### 12.1 Rb

```text
Rb = Usource / Isource
```

分别计算：

- `Rb5_T1`
- `Rb5_T2`
- `Rb5_T3`

### 12.2 Offset

```text
Offset = USG(t3, p0)
```

表示常温/基准温度下零点输出。

### 12.3 Span

```text
Span = USG(t3, p100) - USG(t3, p0)
```

表示满量程输出。

### 12.4 NL / Linearity

```text
slope = (USG_high - USG_low) / (P_high - P_low)
ideal_mid = slope * (P_mid - P_low) + USG_low
NL = (USG_mid - ideal_mid) * 100 / (USG_high - USG_low)
```

用于计算中间压力点相对理想直线的误差。

### 12.5 TCO

```text
TCO = (USG_p0_t2 - USG_p0_t3) / (Temp_t2 - Temp_t3) * 100 / Span_t3
```

表示零位温度系数。

### 12.6 TCS

```text
TCS = ((USG_p100_t2 - USG_p0_t2) - (USG_p100_t3 - USG_p0_t3))
      / (Temp_t2 - Temp_t3) * 100 / Span_t3
```

表示灵敏度温度系数。

### 12.7 TCR

```text
TCR = (Rb_t2 - Rb_t3) / (Temp_t2 - Temp_t3) * 100 / Rb_t3
```

表示电阻温度系数。

### 12.8 THO

```text
THO = (USG_p0_t3 - USG_p0_t1) * 100 / Span_t3
```

表示零位温度迟滞。

### 12.9 THS

```text
THS = ((USG_p100_t3 - USG_p0_t3) - (USG_p100_t1 - USG_p0_t1)) * 100 / Span_t3
```

表示灵敏度温度迟滞。

### 12.10 PH

```text
PH = (USG_p0_t3_R - USG_p0_t3_Start) * 100 / (USG_p100_t3 - USG_p0_t3_R)
```

表示压力回差。

### 12.11 TCT

```text
TCT = (UT_t2 - UT_t3) / (Temp_t2 - Temp_t3)
```

表示温度传感器温度系数。

### 12.12 合格判定

每个指标会查询测试方案中的上下限：

```text
[Specs]
Offset.Min
Offset.Max
Span.Min
Span.Max
...
```

判定规则：

- 值为 NaN 或非法值：Error。
- 超出启用指标的上下限：Warn。
- 在范围内：Ok。
- 指标未启用：仍计算，但不按限值判定。

## 13. 数据保存和导出

### 13.1 实时矩阵导出

`DataMatrix.ExportCsv` 将当前矩阵保存为 CSV：

- 第一列：slot。
- 第二列：SerialNo。
- 后续列：所有出现过的测试列。

### 13.2 兼容旧格式导出

`LegacyCsvExporter` 可按旧版 ASLab 格式导出完整报告和客户报告。

### 13.3 手动批量采集导出

手动页批量采集结果支持导出 CSV，包含：

- 时间。
- 工位。
- Usig。
- UT。
- Usource。
- Isource。

## 14. 断点续测

测试中断或取消时，如果启用断点保存，程序会保存：

- 当前方案名。
- 当前温度点索引。
- 当前压力点索引。
- 探漏是否完成。
- 已生成矩阵列。
- 数据矩阵 CSV。

重新开始时，如果方案匹配，可以从断点继续。

## 15. 异常处理

程序在多个层级处理异常：

- `AsyncRelayCommand`：捕获按钮命令异常，记录日志并弹窗。
- `TaskRunner`：捕获测试动作异常，记录当前命令、错误和连续错误次数。
- `TestRunViewModel`：测试失败后保存数据并关闭烘箱、泄压、关闭电源。
- `App.DispatcherUnhandledException`：捕获 UI 线程未处理异常。
- `TaskScheduler.UnobservedTaskException`：捕获未观察的 Task 异常。
- `AppLog`：后台队列写日志，避免同步磁盘写入阻塞测试。

测试页会显示最后一次错误，便于操作员现场判断故障点。

## 16. 界面操作说明

### 16.1 自动测试

1. 进入实时测试页。
2. 点击开始。
3. 在运行设置窗口选择方案和工位。
4. 确认采集项、压力控制、烘箱、探漏等选项。
5. 启动测试。
6. 查看板卡采集数据矩阵、统计和实时日志。
7. 如需复制数据，在矩阵中选中单元格或区域，按 Ctrl+C，或右键选择复制选中数据。

### 16.2 手动调试

1. 打开采集卡串口。
2. 选择采集卡地址和通道。
3. 执行单项采集或批量采集。
4. 在 TX/RX 面板查看原始报文。
5. 在历史记录中查看操作过程。

### 16.3 配置维护

1. 在配置页选择方案。
2. 编辑压力点、温度点、压力类型和指标限值。
3. 配置设备型号、地址和指令模板。
4. 编辑或导入工位对应表。
5. 保存配置。
